# SkillMapper

A web app that indexes AI agent skills from external repositories, categorizes them, and matches them to user tasks using a hybrid ranking engine.

## Features

- **Skill Search** -- Browse and search 251 AI skills from 29 source repositories
- **Single Task Matching** -- Describe a task and get ranked skill recommendations with confidence scores
- **Multi-Task Matching** -- Submit multiple tasks and get skill recommendations for each
- **Plan Mapping** -- Paste a project plan and get a sequenced skill application roadmap
- **Category Browsing** -- Explore skills across 12 categories (coding, chatbot, DevOps, etc.)
- **Curate** -- Install matched skills into your local Claude Code workspace via [`/curate`](#curate-skills)
- **Admin Dashboard** -- Monitor sync status and trigger data refresh from sources
- **Transparent Scoring** -- See exactly why each skill matched (lexical, semantic, category, popularity)
- **Zod Validation** -- All match API inputs validated with strict schemas

## Quick Start

### Prerequisites

- **Node.js 20 or newer.** Next 15 + Turbopack require Node 18.18+ in theory; 20 LTS is what this project is exercised against. Verify with `node --version`. No `.nvmrc` is pinned yet — track [W2-C](#setup-macro-status) for the eventual `engines` enforcement.
- **npm** (ships with Node).
- No external database needed for local dev — SQLite ships via Prisma.

### Setup

```bash
# 1. Install dependencies
npm install

# 2. Create your local .env from the template (REQUIRED — step 3 fails without it)
cp .env.example .env

# 3. Create database and run migrations (auto-seeds 251 skills via prisma/seed.ts)
npx prisma migrate dev

# 4. Start the dev server
npm run dev
```

Visit [http://localhost:3000](http://localhost:3000).

Until [SOL-467 / W2-C](#setup-macro-status) lands a `npm run setup` macro, steps 1–3 must be run in order. If you skip step 2 you will see `P1012 Environment variable not found: DATABASE_URL` on step 3 — see [Troubleshooting](#troubleshooting).

## Matching Algorithm

The scoring engine combines multiple signals:

```
finalScore = 0.55 * lexical + 0.25 * category + 0.12 * popularity + 0.08 * exactMatch
```

When an embedding provider is configured:
```
finalScore = 0.30 * lexical + 0.40 * semantic + 0.15 * category + 0.10 * popularity + 0.05 * exactMatch
```

Each skill receives a human-readable rationale explaining the match.

A Python reference port lives at [tools/matcher.py](tools/matcher.py) and is parity-tested against the TS scorer on 27 queries × 2 weight configs — see [tools/parity_tests.py](tools/parity_tests.py). The TS oracle is regenerated via `npx tsx tools/parity-runner.ts`.

```bash

  -H 'Content-Type: application/json' \
  -d '{"query":"deploy a docker container to production"}'
```

The body field is `query`, not `task` — earlier versions of this README had the field name wrong.

## Curate Skills

The `/curate` page lets you select matched skills and install them as `SKILL.md` files into a local Claude Code workspace (the default target is `~/.claude/skills/`). The synthesis pipeline lives at [lib/services/skill-installer.ts](lib/services/skill-installer.ts) — `buildSkillMd` is the function under acceptance test for [SOL-519 / M2](#milestones).

Acceptance test (do this before kicking off W2 work that depends on Curate):
1. Run a match.
2. Open `/curate`, select 2 skills, install.
3. Open a fresh Claude Code session and confirm the installed skills load (`/skills` should list them, and a relevant prompt should surface them).

If the skills don't load, the `SKILL.md` format synthesizer is the blocker — file a bug against `lib/services/skill-installer.ts:buildSkillMd` before any subsequent W2 work.

## Environment Variables

Copy [.env.example](.env.example) to `.env` before running migrations.

| Variable | Required | Default | Purpose |
|---|---|---|---|
| `DATABASE_URL` | yes | `file:./dev.db` | SQLite path for local dev; PostgreSQL URL in prod |
| `EMBEDDING_PROVIDER` | no | `none` | `none` \| `openai` \| `local` |
| `OPENAI_API_KEY` | conditional | unset | Required when `EMBEDDING_PROVIDER="openai"` |
| `GITHUB_TOKEN` | no | unset | Not in `.env.example` yet. If unset, `/api/sync/sources` hits GitHub anonymously and is capped at 60 requests/hour. Adding a personal access token (no scopes required for public-repo reads) raises the cap to 5,000/hr — strongly recommended if you'll trigger sync more than once. |

## Troubleshooting

**`P1012 Environment variable not found: DATABASE_URL`** when running `npx prisma migrate dev`
You skipped `cp .env.example .env`. Run it, then retry.

**`npm install` warns about vulnerabilities (1 critical, 9 high, 4 moderate as of writing)**
These are in transitive dependencies of the Next 15 / Turbopack toolchain. They do not block local dev. We track upstream releases — see the `Future TODOs` section. Don't run `npm audit fix --force`; it will downgrade Next.

**GitHub rate limit hit while triggering `/api/sync/sources`**
You ran sync more than ~60 times in an hour without a `GITHUB_TOKEN`. Add one to your `.env` (see [Environment Variables](#environment-variables)). The token only needs `public_repo` read; personal access tokens with no scopes also work for purely public-repo reads.

**`Error: listen EADDRINUSE :::3000`**
Port 3000 is in use. Either kill the existing process (`lsof -ti:3000 | xargs kill`) or run with a different port: `PORT=3001 npm run dev`.

**Prisma deprecation warning about `package.json#prisma`**
Cosmetic. Prisma 7 wants the block moved to `prisma.config.ts`. Tracked separately; safe to ignore for local dev.

**`/curate` installs a skill but it doesn't load in Claude Code**
The synthesized `SKILL.md` format is the suspect. Capture the installed `SKILL.md` and the error from Claude Code, then file against `lib/services/skill-installer.ts:buildSkillMd`. Do not work around — fix the synthesizer.

## Design Decisions

1. **SQLite for MVP** -- Zero-config, instant dev. One-line swap to PostgreSQL via Prisma.
2. **In-memory TF-IDF** -- With 251 skills, in-memory scoring gives full control and runs instantly. Upgrade path to pgvector for scale.
3. **Rule-based plan decomposition** -- No external API dependencies. LLM-assisted decomposition as future enhancement.
4. **Mock data fallback** -- System works without internet. Real scraper tries agent-skills.cc first, falls back to mock data.
5. **Embedding abstraction** -- TF-IDF now, swap to OpenAI/Gemini via env var later.
6. **Multi-source deduplication** -- Skills from multiple data files are merged by slug, preventing duplicates.
7. **Python reference port** -- `tools/matcher.py` is parity-tested against the TS scorer so a non-TS consumer (eval harness, batch scoring) can be added without reimplementing the algorithm.
